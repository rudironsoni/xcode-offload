macOS developer storage

Move Xcode storage off the internal disk without confusing Apple tools.

xcode-storage keeps Xcode and CoreSimulator pointed at their normal paths while APFS sparsebundles carry the heavy data on external storage.

Start setup Read commands

What it solves

Xcode installs, simulator device sets, DerivedData, archives, and CoreSimulator caches can consume hundreds of gigabytes. Moving those folders with symlinks is fragile, especially around CoreSimulator images and simdiskimaged. xcode-storage uses mounted APFS sparsebundles instead.

Apple paths stay normal

Tools still see paths such as ~/Library/Developer/CoreSimulator/Devices and /Library/Developer/CoreSimulator/Caches.

No symlinked mount parents

The mount manager rejects symlinked Apple paths because CoreSimulator can reject them.

Fail closed by default

If a managed path is mounted from an unexpected backend, the tool refuses to detach it.

Repeatable repair

LaunchAgents, LaunchDaemons, helpers, shims, and mount state are managed with repeatable commands.

The short version

export XCODE_STORAGE_ROOT="/Volumes/YourExternalVolume"

xcode-storage repair \
  --root "$XCODE_STORAGE_ROOT" \
  --home "$HOME" \
  --scope user \
  --install-shims \
  --dry-run

xcode-storage repair \
  --root "$XCODE_STORAGE_ROOT" \
  --home "$HOME" \
  --scope user \
  --install-shims \
  --load

sudo xcode-storage daemon install \
  --root "$XCODE_STORAGE_ROOT" \
  --home "$HOME"

xcode-storage doctor \
  --root "$XCODE_STORAGE_ROOT" \
  --require-shims \
  --strict

When to use mounts

Use the mounts group when you want Apple default paths backed directly by APFS sparsebundles: 

xcode-storage mounts install --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --load
sudo xcode-storage mounts install --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope system --load
xcode-storage mounts status --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope all
Do not symlink CoreSimulator Images or Volumes. Use mounted APFS sparsebundles. Xcode and simdiskimaged can reject symlinked mount parents.
