setup

Install the CLI, choose a root, then let launchd do the routine work.

User-owned mounts run without sudo. The system cache helper needs sudo because it manages paths under /Library/Developer.

Install the CLI

Install xcode-offload from the public Homebrew tap. The formula builds from the tagged source release. 

brew install rudironsoni/tap/xcode-offload
xcode-offload version

If you prefer to tap first, run brew tap rudironsoni/tap, then brew install xcode-offload.

Pick a root

Choose the external volume yourself. The tool will not pick one for you. 

export XCODE_OFFLOAD_ROOT="/Volumes/YourExternalVolume"

Preview user changes

xcode-offload repair \
  --root "$XCODE_OFFLOAD_ROOT" \
  --home "$HOME" \
  --scope user \
  --install-shims \
  --dry-run

Install user-owned pieces

xcode-offload repair \
  --root "$XCODE_OFFLOAD_ROOT" \
  --home "$HOME" \
  --scope user \
  --install-shims \
  --load

This installs the user LaunchAgent and optional shims. It does not write root-owned files.

Install the system helper

sudo xcode-offload daemon install \
  --root "$XCODE_OFFLOAD_ROOT" \
  --home "$HOME"

daemon install and launchd install install the same system LaunchDaemon and helper.

Run the doctor

xcode-offload doctor \
  --root "$XCODE_OFFLOAD_ROOT" \
  --require-shims \
  --strict

Use APFS mount mode

For direct mounts at the Apple paths, use the mounts group: 

xcode-offload mounts install --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope user --load
sudo xcode-offload mounts install --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope system --load
xcode-offload mounts status --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope all
Keep the external volume mounted before opening Xcode or Simulator. If the volume is missing, launchd can only log the missing root and try again later.
