Release policy

Releases use SemVer tags and publish a macOS artifact through GitHub Releases.

Version format

Release tags use the form vMAJOR.MINOR.PATCH, for example v0.1.0. The installed tool reports the release version with: 

xcode-storage version

Release notes

User-facing changes are recorded in CHANGELOG.md. GitHub Releases attach the packaged macOS artifact and checksum for the tag.

GitHub automation

The repository uses GitHub Actions for CI, release preparation, release publishing, and this documentation site. The workflows are part of the repository so release behavior is reviewable with the code.

Before upgrading

Run the doctor before and after an upgrade. That gives you a clear before-and-after view of mount state, launchd jobs, and shim health. 

xcode-storage doctor --root "$XCODE_STORAGE_ROOT" --require-shims --strict
