releases

Release policy

Releases use SemVer tags. GitHub Releases publish the macOS artifact and checksum.

Version format

Release tags use vMAJOR.MINOR.PATCH, for example v0.1.0. Check the installed version with: 

xcode-offload version

Release notes

User-facing changes go in CHANGELOG.md. The GitHub Release for each tag includes the packaged macOS artifact and its checksum.

GitHub automation

GitHub Actions runs CI, prepares releases, publishes releases, deploys this docs site, and can open a Homebrew tap update when HOMEBREW_TAP_TOKEN is configured.

Homebrew

The public tap formula builds from the tagged source release. 

brew install rudironsoni/tap/xcode-offload

Before upgrading

Run the doctor before and after an upgrade. That gives you a before-and-after view of mount state, launchd jobs, and shim health. 

xcode-offload doctor --root "$XCODE_OFFLOAD_ROOT" --require-shims --strict
