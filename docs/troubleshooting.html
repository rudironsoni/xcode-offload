debugging

Check the mount state first.

Most problems come from a missing volume, a stale mount, or launchd state. Prove those before deleting simulator data or changing Xcode.

Doctor fails

xcode-offload doctor --root "$XCODE_OFFLOAD_ROOT" --require-shims --strict --json

Use JSON when you need exact labels and details. Use the normal output for quick terminal checks.

Need the full command log

Action commands show concise steps by default. Re-run the same command with --verbose to see both the readable summary and the underlying commands sent to macOS tools. 

sudo xcode-offload mounts repair --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope system --load --verbose
xcode-offload mounts status --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope all --verbose

External root is missing

Mount the external volume, then run the doctor again. The tool will not choose a fallback volume.

Managed path is a symlink

Replace the symlink with a real directory before using mounts install. Symlinked mount parents can fail under CoreSimulator.

Managed path uses another backend

The tool refuses to detach it. Inspect the current mount before deciding what owns it. 

mount | grep CoreSimulator
hdiutil info

CoreSimulator Devices is mounted from a physical external APFS volume

The device path must stay a normal Apple path, but the backend should be the managed DeviceSet.sparsebundle. A raw physical external APFS volume mounted directly at ~/Library/Developer/CoreSimulator/Devices can block CoreSimulator writes even when the mount looks healthy. 

xcode-offload mounts repair --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope user --load
xcode-offload mounts status --root "$XCODE_OFFLOAD_ROOT" --home "$HOME" --scope user
xcode-offload sim reset \
  --name Orlix-iPhone-15-Pro-Max \
  --device-type com.apple.CoreSimulator.SimDeviceType.iPhone-15-Pro-Max \
  --runtime com.apple.CoreSimulator.SimRuntime.iOS-26-5 \
  --verify \
  --screenshot /tmp/orlix-verify.png

The reset command creates a fresh simulator in the currently mounted device store. Prefer that over migrating or copying old simulator directories.

CoreSimulator cache is not mounted

Reload the system daemon, or reinstall it with the current root and home. 

sudo xcode-offload daemon install --root "$XCODE_OFFLOAD_ROOT" --home "$HOME"
xcode-offload doctor --root "$XCODE_OFFLOAD_ROOT" --strict

simctl is slow after a fresh simulator

Recent iOS runtimes can spend several minutes in first-boot migration. sim recreate --boot, sim reset --verify, and sim verify use a long timeout for that reason.

Backups are taking space

xcode-offload never deletes backups automatically. Review old backup directories after the current mount state is healthy. 

find "$XCODE_OFFLOAD_ROOT/Xcode/Backups/mounts" -maxdepth 2 -type f -name '*.manifest' -print
