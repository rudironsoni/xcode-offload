debugging

Prove the mount state first.

Most failures are path, mount, or launchd state issues. Check those before changing Xcode or deleting simulator data.

Doctor fails

xcode-storage doctor --root "$XCODE_STORAGE_ROOT" --require-shims --strict --json

Use JSON when you need exact labels and details. The human output is better for quick terminal checks.

External root is missing

Mount the external volume, then run the doctor again. The tool will not pick a fallback volume for you.

Managed path is a symlink

Replace the symlink with a real directory before using mounts install. Symlinked mount parents can fail under CoreSimulator.

Managed path uses another backend

The tool refuses to detach it. Inspect the current mount and decide whether it belongs to another setup. 

mount | grep CoreSimulator
hdiutil info

CoreSimulator cache is not mounted

Reload the system daemon or reinstall it with the current root and home. 

sudo xcode-storage daemon install --root "$XCODE_STORAGE_ROOT" --home "$HOME"
xcode-storage doctor --root "$XCODE_STORAGE_ROOT" --strict

simctl is slow after a fresh simulator

Recent iOS runtimes can spend several minutes in first-boot migration. sim recreate --boot defaults to a long timeout for that reason.

Backups are taking space

xcode-storage never deletes backups automatically. Review and remove old backup directories yourself after confirming the current mount state is healthy. 

find "$XCODE_STORAGE_ROOT/Xcode/Backups/mounts" -maxdepth 2 -type f -name '*.manifest' -print
