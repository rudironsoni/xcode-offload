Common operations

These flows cover install, verification, Apple-tool checks, and rollback.

Install on a new machine

  1. Mount the external volume.
  2. Set XCODE_STORAGE_ROOT to a path on that volume.
  3. Run repair --dry-run and read the plan.
  4. Run user repair with --load.
  5. Install the system daemon with sudo.
  6. Run doctor --strict.
export XCODE_STORAGE_ROOT="/Volumes/YourExternalVolume"
xcode-storage repair --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --install-shims --dry-run
xcode-storage repair --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --install-shims --load
sudo xcode-storage daemon install --root "$XCODE_STORAGE_ROOT" --home "$HOME"
xcode-storage doctor --root "$XCODE_STORAGE_ROOT" --require-shims --strict

Enable APFS mount mode

xcode-storage mounts install --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --dry-run
xcode-storage mounts install --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --load
sudo xcode-storage mounts install --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope system --load
xcode-storage mounts status --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope all

Verify in a scratch root

xcode-storage mounts verify \
  --scratch-root "/Volumes/YourExternalVolume/xcode-storage-verify" \
  --mode user

This creates disposable sparsebundles, mounts them, checks status, unmounts them, and removes the scratch run directory.

Run Apple tools without shims

PATH="/usr/bin:/bin:/usr/sbin:/sbin" /usr/bin/xcrun simctl list devices available
PATH="/usr/bin:/bin:/usr/sbin:/sbin" /usr/bin/xcodebuild -version

If APFS mount mode is working, these commands should see the mounted Apple paths without any shim involvement.

Back out user mount mode

xcode-storage mounts uninstall --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope user --unload

Back out system mount mode

sudo xcode-storage mounts uninstall --root "$XCODE_STORAGE_ROOT" --home "$HOME" --scope system --unload
