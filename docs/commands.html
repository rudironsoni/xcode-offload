Command reference

Use --dry-run before a command changes mount state, launchd jobs, or files under Apple developer paths. Add --verbose when you need raw commands or the full mount-check list.

Output behavior

Commands that change files, mounts, or launchd jobs print concise human-readable steps by default. They do not print every mkdir, hdiutil, chmod, or launchctl call unless --verbose is present.

mounts status also stays concise by default. Use mounts status --verbose when you need every PASS, WARN, and FAIL check with exact paths and mount details.

Health checks

xcode-offload doctor [--root PATH] [--require-shims] [--skip-simctl] [--strict] [--json]

doctor checks directories, sparsebundles, mounts, launchd jobs, and optional shims. --strict also checks APFS state, backend provenance, plist validity, and launchd exit status.

Repair

xcode-offload repair [--root PATH] [--home PATH] [--shim-dir PATH] [--scope user|system|all] [--install-shims] [--load] [--dry-run] [--verbose]

repair creates the storage root if needed, mounts the device and cache stores, installs launchd jobs, and can install shims.

Initialize storage

xcode-offload init [--root PATH] [--dry-run] [--no-create-images] [--verbose]

init creates the directory layout and sparsebundles. Use --no-create-images when you only want the directory layout.

Mount legacy stores

xcode-offload mount devices|caches [--root PATH] [--dry-run] [--verbose]
xcode-offload unmount devices|caches [--root PATH] [--dry-run] [--verbose]

These commands manage the device and cache sparsebundles used by the repair flow.

Install shims

xcode-offload install-shims [--root PATH] [--shim-dir PATH] [--dry-run] [--verbose]

Shims wrap xcrun, simctl, and xcodebuild for explicit flag routing. They are optional when APFS mount mode is active.

Launchd

xcode-offload daemon install [--root PATH] [--home PATH] [--no-load] [--dry-run] [--verbose]
xcode-offload launchd install [--root PATH] [--home PATH] [--no-load] [--dry-run] [--verbose]
xcode-offload install-launchd [--root PATH] [--home PATH] [--scope user|system|all] [--load] [--dry-run] [--verbose]
xcode-offload uninstall-launchd [--root PATH] [--home PATH] [--scope user|system|all] [--unload] [--dry-run] [--verbose]

daemon install and launchd install are the normal names for the system helper install. install-launchd and uninstall-launchd remain available for lower-level launchd work.

APFS mount manager

xcode-offload mounts install [--root PATH] [--home PATH] [--scope user|system|all] [--load] [--dry-run] [--verbose]
xcode-offload mounts repair [--root PATH] [--home PATH] [--scope user|system|all] [--load] [--dry-run] [--verbose]
xcode-offload mounts uninstall [--root PATH] [--home PATH] [--scope user|system|all] [--unload] [--dry-run] [--verbose]
xcode-offload mounts status [--root PATH] [--home PATH] [--scope user|system|all] [--json] [--verbose]
xcode-offload mounts verify --scratch-root PATH [--mode user|system|e2e] [--home PATH] [--runtime ID] [--device-type ID] [--keep-artifacts] [--allow-system] [--allow-sim-delete]

xcodes compatibility

xcode-offload xcodes install-profile [--root PATH] [--home PATH] [--load] [--dry-run] [--verbose]
xcode-offload xcodes doctor [--root PATH] [--home PATH] [--require-xcodes] [--strict] [--json]
xcode-offload xcodes env install [--root PATH] [--home PATH] [--directory PATH] [--dry-run] [--verbose]

xcodes install-profile prepares APFS-backed Apple paths for xcodes and standard Xcode tools. It does not require command shims.

Simulator helpers

xcode-offload sim runtimes
xcode-offload sim devices [--all]
xcode-offload sim recreate --name NAME --device-type TYPE --runtime RUNTIME [--boot] [--boot-timeout SECONDS]
xcode-offload sim reset --name NAME --device-type TYPE --runtime RUNTIME [--boot] [--verify] [--boot-timeout SECONDS] [--screenshot PATH]
xcode-offload sim verify (--name NAME | --udid UDID) [--boot-timeout SECONDS] [--screenshot PATH]
xcode-offload sim open (--name NAME | --udid UDID) [--boot-timeout SECONDS]

sim recreate deletes an existing simulator with the same name, creates a fresh one, and can boot it. First boot can be slow on recent iOS runtimes, so the boot timeout is intentionally long. sim reset --verify recreates a named simulator, boots it, waits for bootstatus, runs a command inside it, and writes a screenshot. sim verify performs the same usability proof for an existing simulator by name or UDID. sim open is idempotent: a simulator that is already booted is treated as ready, then Simulator is opened for that device.
